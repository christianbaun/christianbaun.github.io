Drones with Artificial Intelligence - Master Project - Study Field Intelligent Systems (WS2526)

Participants in the project form groups of up to four people. Each group is provided with an AI-capable FPV drone and the necessary components. Their tasks are the development, implementation, investigation, and evaluation of exciting and comprehensible drone AI applications.

  • Task 1: Become familiar with the FPV drone and the additional hardware components (single-board computer, camera, A/V video grabber, AI accelerator, etc.).
    • Learn about the capabilities and limitations of the hardware components. These include:
      • Frame, flight controller and electronic speed controller, FPV transmitter (for video), ELRS receiver, motors, GPS receiver, camera(s), batteries, remote control, single-board computer (e.g. Raspberry Pi Zero 2 WH), AI accelerator (e.g. Google Coral USB Accelerator), FPV goggles, A/V video grabber, etc.
    • Learn about the capabilities and limitations of the software components. These include:
      • Flight controller firmware (Betaflight, INAV, ArduPilot), ground control station (e.g. QGroundControl for ArduPilot), operating system (e.g. Raspberry Pi OS, Ubuntu) for the single-board computer, AI software (e.g. TensorFlow Lite, YOLO), etc.
      • The focus of the project is on the AI applications (see Task 2)! Building a new drone is neither necessary nor the goal of the project. Modifications to the existing drones are possible if desired.
  • Task 2: Develop, integrate, and test AI application(s). Possible application areas include:
    • Computer Science: Object recognition (e.g. detection of people and animals)
    • Logistics: Automated delivery and pickup of objects
    • Geography: Biodiversity research or detection of storm damage
    • Engineering: Structural inspections
    • Agricultural Sciences: Monitoring of plants and pest infestation
    • Archaeology: Aerial imaging
    • Atmospheric Research: Smog measurements
  • Task 3: Research and integrate autopilot options.
    • For example using INAV and ArduPilot.
  • Task 4: Research and integrate delivery/payload options.
    • Develop and test a simple drop mechanism (e.g. using a servo).
  • Task 5: Documentation and presentation of the results of Tasks 1–4.
    • You will develop documentation and instructions that enable students, researchers, and lecturers to reproduce the AI drone scenarios and use them for their own modules and research projects.
    • There will be no slide presentations or PDF project reports! Each team develops a complete and comprehensible online documentation (e.g. via GitHub Pages) and presents their results with a poster and a live demonstration.

Equipment

Each team receives the following equipment:

  • 3.5'' FPV drone with CineWhoop frame (including propeller guards), fully assembled and ready to fly
  • Skyzone Cobra X FPV goggles
  • Radiomaster Boxer ELRS transmitter (remote controller) with EdgeTX firmware v2.11.3 and binding phrase configured to match the receiver: drone[1-3]
  • Google Coral USB Accelerator AI accelerator
  • Raspberry Pi Zero 2 WH single-board computer
    • A Raspberry Pi OS Lite (32-bit) based on Debian 11.11 (Bullseye) is installed on the SD card. With this version, the libraries required for the Google Coral USB Accelerator and the necessary Python version can be installed very easily.
  • Raspberry Pi Camera Module v2
  • Mini-HDMI to standard HDMI cable
  • Micro-USB (m) to USB Type A (f) cable
  • USB audio/video grabber (MacroSilicon MS210x)
  • Smoke stopper (short-circuit protection plug)
  • SkyRC B6neo+ charger
  • Allen and hex keys in the sizes commonly used for FPV drones
  • Spare propellers (3.5'')
  • USB card reader for SD and microSD memory cards
